# Server-Side Caching Configuration — FMIKOM Portal

To achieve the best possible load times and maximize the Lighthouse Performance score (aiming for 90+), static assets generated by Vite and standard web resources (fonts, images, media) should be cached aggressively.

---

## 1. Nginx Configuration

In your production Nginx server block, configure cache-control headers for Vite static assets. Because Vite appends unique hash keys to compiled asset filenames (e.g. `app-C3d89A1b.js`), these files are **immutable** and can be cached indefinitely on client browsers and intermediary CDNs without risk of outdated code.

Add the following location block to your Nginx virtual host configuration:

```nginx
# Cache Vite built assets (JS, CSS, fonts, etc.)
location ~ ^/build/assets/ {
    access_log off;
    log_not_found off;
    expires max;
    
    # Enforce aggressive browser and CDN caching
    add_header Cache-Control "public, max-age=31536000, immutable";
    
    # CORS headers for assets accessed via subdomains
    add_header Access-Control-Allow-Origin "*";
    
    # Pre-compressed asset support (if using gzip/brotli pre-compression)
    gzip_static on;
    brotli_static on;
}

# Cache user storage uploads (Images, Avatars, Banners)
location ~ ^/storage/ {
    access_log off;
    log_not_found off;
    expires 30d;
    
    # 30-day cache control with revalidation
    add_header Cache-Control "public, max-age=2592000, must-revalidate";
}
```

## 2. Cloudflare CDN Configuration

If you are using Cloudflare, you can configure Page Rules or Cache Rules to offload bandwidth and decrease latency globally.

### Cache Rules (Recommended)
Create a new Cache Rule in the Cloudflare Dashboard under **Caching > Cache Rules**:

1. **Rule Name**: `Cache Static Assets & Storage`
2. **If incoming requests match... (Field Expression)**:
   - `URI Path` starts with `/build/assets/` OR
   - `URI Path` starts with `/storage/`
3. **Then... (Cache Eligibility)**:
   - Select **Eligible for cache**
4. **Edge TTL**:
   - Set Edge TTL to **Respect origin headers** (or force a specific duration, e.g., 1 month).
5. **Browser Cache TTL**:
   - Set to **Respect origin headers**.

### Cloudflare Polish & Brotli
Ensure the following are active under the **Speed** menu:
- **Brotli compression**: Enabled (gives ~15-20% smaller payload sizes compared to gzip).
- **Polish**: Enabled (lossy/lossless) to auto-convert uploaded images to WebP/AVIF formats dynamically at the edge.
